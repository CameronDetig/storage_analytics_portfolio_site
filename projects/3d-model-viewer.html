3D Model Viewer

WebGL-Based Interactive 3D Rendering

Project Overview

This project is a WebGL-based 3D model viewer built from scratch using vanilla JavaScript and WebGL. It demonstrates fundamental computer graphics concepts including 3D transformations, lighting models, and interactive camera controls—all implemented without relying on high-level 3D libraries like Three.js.

The viewer was developed as part of my computer graphics coursework, where I learned the low-level details of how 3D rendering works in the browser. Building everything from the ground up gave me a deep understanding of the graphics pipeline, shader programming, and real-time rendering techniques.

Key Features

  • Interactive 3D Rendering - Real-time WebGL rendering with smooth frame rates
  • Phong Lighting Model - Point light source with ambient, diffuse, and specular components
  • Trackball Camera Controls - Intuitive mouse-based camera rotation and navigation
  • Custom Shaders - Hand-written GLSL vertex and fragment shaders
  • No External Libraries - Built entirely with vanilla JavaScript and WebGL APIs

Technical Implementation

Graphics Pipeline

The viewer implements the full WebGL graphics pipeline from scratch:

  • Vertex Processing - Model-view-projection matrix transformations in vertex shader
  • Rasterization - WebGL handles conversion of 3D geometry to screen pixels
  • Fragment Shading - Phong lighting calculations per-pixel in fragment shader
  • Frame Buffer - Final rendered output to HTML5 canvas element

Lighting Model

The Phong lighting model creates realistic lighting effects by combining three components:

  • Ambient - Base lighting that illuminates all surfaces equally
  • Diffuse - Directional lighting based on surface normal and light direction
  • Specular - Shiny highlights based on viewing angle and light reflection

Camera Controls

The trackball camera system allows intuitive 3D navigation:

  • Rotation - Click and drag to rotate the model in any direction
  • Smooth Motion - Quaternion-based rotations prevent gimbal lock
  • View Matrix - Real-time camera transformation updates

Learning Outcomes

Building this viewer from scratch taught me:

  • How the WebGL graphics pipeline works at a low level
  • Writing and debugging GLSL shader code
  • 3D mathematics including matrix transformations and vector operations
  • Performance optimization for real-time rendering
  • The fundamentals that power modern 3D engines and frameworks
